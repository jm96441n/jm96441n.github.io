JavaScript vs Ruby

This past week has been spent working on a new language, JavaScript. In many ways JavaScript and Ruby are very similar, but deep down they are very different. Today we will be looking at one of the differences, and that is how you loop in each language. Let's look at some of the way's we have to loop in each language, and then compare them. First off let's look at Ruby: .each .each_index .each_slice .each_key .each_value for __ in ___ loop do while .times These are just some of the major way's of looping in Ruby. Most of them use each in some way. The way that these behave is by going through each element (or index, slice, etc whichever the loop specifies) in the span you're looping through. Let's look at some of the ways JavaScript uses: for (var __ = __; [condition]; [incrementExpression]) statement do statement while (condition); while(condition) statement So as we see with JavaScript loops, generally rely on using counter variables, or setting conditions for the loop to run for until it completes the condition. As you can see a big difference between the two is in the way they loop. Both can perform the same loops, but in different matters. For example, let's say we have an array of numbers that we want to find the sum of. In Ruby we would iterate through each element and add them together, whereas in JavaScript we would use a for loop to iterate through the indexes and then add them all together. Let's see that in action: RUBY: array = [1,2,3,4,5,6] sum = 0 array.each{|num| sum+= num } return sum end JAVASCRIPT: var array = [1,2,3,4,5,6]; var sum = 0; for (var i = 0; i <= array.length; i++){ sum += array[i]; }; return sum; If we run this code, we will get the same answer, but if you look at the way the languages handle the loop you will see they go about it in different ways.
